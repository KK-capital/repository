Issue #1: Null Pointer Exception 風險 — 未檢查 productDetail 是否為 null

項目內容
檔案名稱src/main/java/com/cec/oms/service/impl/ProductServiceImpl.java
行數Line 287
修正原因productDao.findById() 可能回傳 null，直接呼叫 .getWeight() 會導致 NPE，在生產環境可能造成整個 API 崩潰。

調整前程式碼：

ProductDetail productDetail = productDao.findById(productId);
BigDecimal weight = productDetail.getWeight();
if (weight.compareTo(maxWeight) > 0) {
    throw new BusinessException("商品重量超過限制");
}

調整後程式碼：

ProductDetail productDetail = productDao.findById(productId);
if (productDetail == null) {
    throw new BusinessException("找不到商品，商品ID: " + productId);
}
BigDecimal weight = productDetail.getWeight();
if (weight != null && weight.compareTo(maxWeight) > 0) {
    throw new BusinessException("商品重量超過限制");
}

Issue #2: N+1 查詢問題 — 迴圈內執行 DB 查詢

項目內容
檔案名稱src/main/java/com/cec/oms/service/impl/ProductServiceImpl.java
行數Line 312-318
修正原因在 for 迴圈內逐筆呼叫 categoryProductDao.existProductOverWeight()，當商品數量大時會產生大量 DB 查詢，嚴重影響效能。應改為批次查詢。

調整前程式碼：

List<String> overWeightProducts = new ArrayList<>();
for (ProductDTO product : productList) {
    boolean isOverWeight = categoryProductDao.existProductOverWeight(
        product.getCategoryId(), product.getWeight()
    );
    if (isOverWeight) {
        overWeightProducts.add(product.getProductId());
    }
}

調整後程式碼：

// 批次查詢，避免 N+1 問題
List<String> productIds = productList.stream()
    .map(ProductDTO::getProductId)
    .collect(Collectors.toList());
List<String> overWeightProducts = categoryProductDao
    .findOverWeightProductIds(productIds);

Issue #3: MyBatis foreach 未檢查空陣列

項目內容
檔案名稱src/main/resources/mapper/CategoryProductMapper.xml
行數Line 45
修正原因MyBatis <foreach> 若傳入 null 或空陣列，會產生語法錯誤的 SQL（IN ()），必須在外層加上 <if> 判斷。

調整前程式碼：

<select id="findOverWeightProductIds" resultType="string">
    SELECT product_id FROM category_product
    WHERE product_id IN
    <foreach collection="productIds" item="id" open="(" close=")" separator=",">
        #{id}
    </foreach>
    AND weight > max_weight
</select>

調整後程式碼：

<select id="findOverWeightProductIds" resultType="string">
    SELECT product_id FROM category_product
    <where>
        <if test="productIds != null and productIds.size() > 0">
            product_id IN
            <foreach collection="productIds" item="id" open="(" close=")" separator=",">
                #{id}
            </foreach>
            AND weight > max_weight
        </if>
    </where>
</select>

Issue #4: Magic Number — 重量上限應使用常數定義

行數Line 295

調整前程式碼：

調整後程式碼：

private static final BigDecimal MAX_PRODUCT_WEIGHT = new BigDecimal("30.0");

// 使用處
BigDecimal maxWeight = MAX_PRODUCT_WEIGHT;
